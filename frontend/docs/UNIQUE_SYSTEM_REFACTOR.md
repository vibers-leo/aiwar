# 유니크 시스템 개편 및 신화 등급 도입 계획 (Unique System Refactor & Mythic Tier)

## 1. 개요 (Overview)
본 문서는 기존 게임 내 최상위 등급이었던 **'유니크(Unique)'의 의미를 재정의**하고, 새로운 최상위 등급인 **'신화(Mythic)'를 도입**하는 대규모 리팩토링 및 기능 개편 계획을 기술합니다.

## 2. 핵심 변경 사항 (Key Changes)

### A. 등급 체계 변경 (Tier System Update)
기존의 `Unique` 등급 명칭을 `Mythic`으로 변경하고, 역할은 그대로 유지(또는 강화)합니다.

*   **변경 전:** 일반(Common) - 희귀(Rare) - 영웅(Epic) - 전설(Legendary) - **유니크(Unique)** - 군단장(Commander)
*   **변경 후:** 일반(Common) - 희귀(Rare) - 영웅(Epic) - 전설(Legendary) - **신화(Mythic)** - 군단장(Commander)

**신화(Mythic) 등급의 특징:**
*   **획득처:** 전설 등급 카드 합성을 통해 획득 가능.
*   **성능:** 군단장 등급에 준하는 강력한 스텟 보유.
*   **코드 변경:** 기존 코드에서 `rarity === 'unique'` 로직을 모두 `rarity === 'mythic'`으로 마이그레이션.

### B. '유니크(Unique)'의 개념 재정의 (Unique Concept Redefinition)
'유니크'는 이제 **카드의 등급(Rarity)이 아닌, 카드의 특별한 상태(Customized State)**를 의미합니다.

*   **개념:** "세상에 단 하나뿐인 나만의 카드"
*   **기능:** 플레이어가 보유한 카드의 **이미지, 영상, 사운드, 이름, 설명**을 직접 커스터마이징할 수 있는 시스템.
*   **대상:** **모든 등급의 카드** (일반 ~ 신화)
    *   예: 내가 아끼는 '일반' 등급 카드를 '유니크'화 하여 나만의 짤방이나 영상을 넣을 수 있음.
    *   예: 강력한 '신화' 등급 카드를 '유니크'화 하여 더 멋지게 꾸밀 수 있음.
*   **스텟 영향:** 유니크화 한다고 해서 기본 전투 스텟이 변하는 것은 아님 (단, '소장 가치' 또는 별도의 '애정 보너스' 고려 가능).

## 3. 상세 구현 계획 (Implementation Plan)

### 단계 1: 등급 명칭 변경 및 마이그레이션 (Phase 1: Rename to Mythic)
기존 시스템의 '유니크'를 '신화'로 안전하게 변경합니다.

1.  **타입 정의 수정 (`types.ts`):** `Rarity` 타입에서 `'unique'`를 `'mythic'`으로 변경.
2.  **데이터 마이그레이션:** DB 및 로컬스토리지에 저장된 기존 `'unique'` 등급 카드를 `'mythic'`으로 일괄 업데이트.
3.  **UI/텍스트 변경:** 게임 내 모든 "유니크" 텍스트를 "신화"로 변경 (영어: Unique -> Mythic). 색상은 기존 유니크 색상(Red/Pink 계열) 유지 또는 신화에 맞는 색상(Celestial Blue 등)으로 변경 고려.
4.  **합성 로직 수정:** 전설 + 전설 + 전설 = **신화** 카드가 나오도록 수정.

### 단계 2: 신 플래그(Flag) 도입 - `isUnique` (Phase 2: New Unique Flag)
카드가 '유니크(커스터마이징 된)' 상태인지 판별하는 속성을 추가합니다.

1.  **카드 스키마 확장:**
    ```typescript
    interface InventoryCard {
        // ... 기존 필드
        rarity: 'common' | 'rare' | 'epic' | 'legendary' | 'mythic' | 'commander'; // 등급
        
        isUnique: boolean; // 유니크화 여부 (신설)
        
        // 유니크화 되었을 때 사용하는 커스텀 데이터 (신설)
        customData?: {
            customName?: string;       // 내가 지은 이름
            customDescription?: string;// 내가 쓴 설명
            customImage?: string;      // 내가 올린 이미지 URL
            customVideo?: string;      // 내가 올린 비디오 URL
            customSound?: string;      // 호버 사운드 등
            createdDate: number;       // 유니크화 한 날짜
        };
    }
    ```
2.  **디스플레이 로직 수정:** `GameCard.tsx` 등에서 `isUnique`가 `true`일 경우, 원본 데이터 대신 `customData`의 내용을 우선적으로 표시.

### 단계 3: '유니크 생성' 메뉴 리뉴얼 (Phase 3: Unique Creation UI)
현재의 '유니크 생성(Unique Create)' 페이지를 **'유니크 제작소(Unique Studio)'**로 전면 개편합니다.

1.  **진입 흐름:** 메뉴 -> 유니크 제작
2.  **기능:**
    *   **베이스 카드 선택:** 인벤토리의 모든 카드 중 하나 선택.
    *   **커스터마이징 툴:** 이미지 업로드, 이름 변경, 설명 작성, (가능하다면) 테두리 효과 선택 등.
    *   **제작 비용:** 일정량의 재화(토큰, 코인) + (옵션) 특수 재료 소모.
3.  **결과:** 선택한 카드가 `isUnique: true`가 되고 커스텀 데이터가 입혀짐. 원본 등급과 레벨은 유지됨.

## 4. 기대 효과 (Expected Outcome)
*   **등급 체계의 확장성:** 신화 등급 도입으로 최상위 콘텐츠(End-game content) 확보.
*   **사용자 애착 강화:** 자신이 좋아하는 카드를 직접 꾸밈으로써 수집 욕구와 애착도 증가.
*   **콘텐츠 재활용:** 낮은 등급의 카드라도 '유니크화'를 통해 버리지 않고 활용할 수 있는 가치 부여.
