### 4. 실제 코드 최적화 적용

#### a) Promise.all 패턴 (CRITICAL)
**파일:** `app/admin/card-assets/page.tsx`
```typescript
// 개선 전: 순차 실행 (~400ms)
const factionsSnapshot = await getDocs(factionsRef);
const cardsSnapshot = await getDocs(cardsRef);

// 개선 후: 병렬 실행 (~200ms) - 50% 개선
const [factionsSnapshot, cardsSnapshot] = await Promise.all([
  getDocs(collection(db, 'factions')),
  getDocs(collection(db, 'cards'))
]);
```

#### b) 동적 모달 임포트 (CRITICAL)
**파일:** `lib/dynamic-modals.tsx`
- 15개 모달 컴포넌트를 lazy loading으로 변경
- 초기 번들 크기 ~100KB 절약
- 프리로드 기능 포함 (`preloadModal`)

#### c) 성능 유틸리티 라이브러리
**파일:** `lib/performance-utils.ts`
- `useReducedMotion()` - 모션 감소 감지
- `MemoCache` - LRU 캐시
- `debounce()`, `throttle()` - 함수 호출 최적화
- `createLookupSet()`, `createLookupMap()` - O(1) 조회

#### d) CSS 접근성 및 성능 개선
**파일:** `app/globals.css`
```css
/* Reduced Motion 지원 */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}

/* 키보드 포커스 상태 */
:focus-visible {
  outline: 2px solid var(--primary-blue);
}

/* Content Visibility 가상화 */
.card-list-item {
  content-visibility: auto;
  contain-intrinsic-size: 0 240px;
}
```

#### e) SEO 메타데이터 개선
**파일:** `app/layout.tsx`
- 키워드 추가
- OpenGraph 메타데이터

#### f) 카드 목록 최적화

## 🔑 핵심 배운 점

1. **Promise.all은 필수** - 독립적인 async 작업은 항상 병렬 실행
2. **Dynamic Import 활용** - 모달처럼 즉시 필요없는 컴포넌트는 lazy load
3. **접근성은 성능** - `prefers-reduced-motion` 지원은 CPU도 절약
4. **Content Visibility** - 긴 목록은 CSS 한 줄로 가상화 가능
