## 1. 💎 토큰 시스템 분석

### 1.1 토큰 획득 경로

| 방법 | 획득량 | 빈도 | 일일 예상 |
|-----|--------|------|----------|
| **시간당 자동 충전** ⭐ | 100 토큰/60분 | 자동 | 2,400 토큰 |
| 신규 가입 보너스 | 1,000 | 1회성 | - |
| 실시간 PVP 승리 | +100 | 무제한 | 변동 |
| 군단 구독 보너스 | 충전량 +10~50, 속도 +10~30분 | 구독 중 | 구독별 |

> **자동 충전 시스템** (`checkAndRechargeTokens` in `firebase-db.ts`)
> - 기본 주기: 60분 (구독 시 최소 10분까지 단축)
> - 기본 충전량: 100 토큰/주기 (구독 시 추가)
> - 최대 보유량: 1,000 (구독 시 +500~2000)

### 1.2 토큰 소모 경로

| 활동 | 비용 | 비고 |
|-----|-----|------|
| 스토리 스테이지 입장 | 50 | 일반 단계 |
| 스토리 보스 입장 | 100 | 보스 단계 |
| AI 대전 입장 (PVP) | 50~150+ | 계단식 증가 (50n) |
| 실시간 PVP 참가 | 50 | 고정 |
| 토큰 팩 구매 | 1,000 | 카드 1장 |

### 1.3 ⚠️ 문제점: 토큰 고갈

```
예시 시나리오 (신규 유저, 1일차):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
초기 토큰: 1,000

- 스토리 5스테이지 클리어: -250 (50 × 5)
- AI 대전 3회: -300 (50 + 100 + 150)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
남은 토큰: 450 → 토큰 부족으로 진행 불가 😢
```

### 1.4 권장 개선안

1. **일일 토큰 충전**: 매일 오전 6시에 100~200 토큰 무료 지급
2. **퀘스트 보상 토큰 추가**: 일일/주간 퀘스트 완료 시 토큰 보상
3. **스토리 무료 플레이**: 첫 클리어는 토큰 무료, 재도전시에만 비용 부과
4. **광고 시청 보상**: 광고 1회 시청 시 토큰 50~100 지급

## 3. ⭐ 경험치 및 레벨 시스템

### 3.1 레벨업 공식

```typescript
레벨 = Math.floor(경험치 / 100) + 1
```

| 레벨 | 필요 총 경험치 | 레벨업 보상 (코인) |
|------|---------------|------------------|
| 1 → 2 | 100 | 200 |
| 2 → 3 | 200 | 300 |
| 5 → 6 | 500 | 600 (+ 카드 1장) |
| 10 → 11 | 1,000 | 1,100 (+ 카드 1장) |

### 3.2 경험치 획득

| 활동 | 경험치 |
|-----|--------|
| PVP 승리 (tactics) | 50 |
| PVP 승리 (strategy) | 70 |
| PVP 패배 | 10 |
| 스토리 클리어 | 스테이지별 상이 |

### 3.3 ✅ 평가: 양호

레벨업 속도와 보상이 적절합니다.

## 4. ⚔️ 스토리 난이도 분석

### 4.1 챕터별 적 파워 분포

| 챕터 | 스테이지 | 난이도 | 적 평균 파워 | 권장 플레이어 파워 |
|-----|---------|-------|------------|-----------------|
| 1 | 1-1 ~ 1-5 | EASY/NORMAL | 30~50 | 35~55 |
| 1 | 1-6 ~ 1-9 | NORMAL/HARD | 50~70 | 55~75 |
| 1 | 1-10 | BOSS | 80+ | 85+ |
| 2 | 2-1 ~ 2-10 | NORMAL~BOSS | 60~100 | 65~105 |
| 3 | 3-1 ~ 3-10 | HARD~BOSS | 80~120 | 85~125 |

### 4.2 ⚠️ 난이도 곡선 문제

```
   파워
    ▲
120 │                              ●
100 │                         ●━━━━●
 80 │                    ●━━━━●
 60 │               ●━━━━●
 40 │          ●━━━━●
 20 │     ●━━━━●
  0 └────────────────────────────────▶ 스테이지
       1-1   1-5   1-10  2-5   2-10  3-10
       
⚠️ 1-10 → 2-1 사이의 급격한 상승
⚠️ 챕터 3 진입 시 난이도 점프
```

### 4.3 권장 개선안

1. 챕터 간 난이도 전환을 더 완만하게
2. 챕터 클리어 보상으로 강력한 카드 지급
3. "연습 모드" 추가 (토큰 비용 없이 연습 가능)

## 5. 🔄 실시간 반영 점검

### 5.1 Firebase 동기화 경로

```
┌─────────────────────────────────────────────────────┐
│  UserContext Provider                               │
├─────────────────────────────────────────────────────┤
│                                                     │
│  coins, tokens, level, experience, inventory        │
│         ↑                                           │
│    onSnapshot() ─────── Firestore                   │
│         │                                           │
│    addCoins() ──────▶ updateDoc()                   │
│    addTokens() ─────▶ updateDoc()                   │
│    addExperience() ─▶ updateDoc()                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 5.2 검증된 동기화 함수들

| 함수 | 위치 | 동작 |
|-----|------|------|
| `addCoins()` | UserContext | Firebase 직접 업데이트 ✅ |
| `addTokens()` | UserContext → `updateTokens()` | Firebase 직접 업데이트 ✅ |
| `addExperience()` | UserContext | Firebase 직접 업데이트 ✅ |
| `refreshData()` | UserContext | 전체 프로필 재로드 ✅ |

### 5.3 ✅ 평가: 양호

모든 재화 변경은 Firebase에 즉시 반영되며, UI도 Context를 통해 즉시 업데이트됩니다.

## 6. 📋 권장 조치 사항

### 🔴 긴급 (즉시 수정 권장)

| 항목 | 현재 | 권장 |
|-----|-----|------|
| 일일 무료 토큰 | 없음 | 100~200 토큰 매일 지급 |
| 스토리 첫 클리어 | 토큰 50 소모 | 무료 (재도전시 50) |

### 🟡 중요 (1주 내 수정 권장)

| 항목 | 현재 | 권장 |
|-----|-----|------|
| AI 대전 토큰 비용 | 계단식 (50n) | 고정 50 또는 상한선 200 |
| 퀘스트 토큰 보상 | 없음 | 일일 퀘스트 완료 시 +50 |

### 🟢 개선 (여유 시 수정)

| 항목 | 현재 | 권장 |
|-----|-----|------|
| 챕터 간 난이도 | 급격한 상승 | 완만한 곡선 |
| 연습 모드 | 없음 | 토큰 무비용 연습 모드 추가 |
